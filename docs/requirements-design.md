## 一、需求背景

### 1.1 项目定位

Build-Eye 是一个 **CI 构建监测与根因分析系统**，专为 vLLM-Ascend 项目设计，旨在：
- 自动化分析 CI 构建失败原因
- 提供智能分类和修复建议
- 生成可视化报表和趋势分析

### 1.2 需求清单

| 需求编号 | 需求名称 | 需求描述 | 优先级 |
|---------|---------|---------|--------|
| REQ-001 | AI 智能分类 | 基于 ML 模型优化失败根因分类精度 | P0 |
| REQ-002 | 智能告警通知 | AI 驱动的多渠道告警（钉钉/飞书/邮件） | P0 |
| REQ-003 | 趋势预测分析 | AI 分析历史数据，预测失败趋势 | P1 |
| REQ-004 | 自动修复建议 | AI 生成可执行的修复代码建议 | P1 |
| REQ-005 | 异常检测 | AI 识别偏离正常模式的异常构建 | P2 |
| **REQ-006** | Nightly 流水线支持 | 分析代码仓中的 nightly 定时流水线结果 | **P0** |
| **REQ-007** | Weekly 流水线支持 | 分析 weekly 冒烟测试流水线结果 | **P1** |
| **REQ-008** | 可视化报表 | 提供多维度数据可视化仪表盘 | **P0** |

### 1.3 流水线类型扩展

| 流水线类型 | 触发方式 | 分析重点 | 数据粒度 |
|-----------|---------|---------|---------|
| **PR 流水线** | PR 提交/更新 | 代码变更影响、回归检测 | 每个 PR |
| **Nightly 流水线** | 定时（每日） | 完整测试覆盖、环境稳定性 | 每日汇总 |
| **Weekly 流水线** | 定时（每周） | 全面冒烟、集成测试 | 每周汇总 |
| **手动触发** | 人工触发 | 按需验证、特定场景 | 单次执行 |

### 2.2 Nightly 流水线支持（REQ-006）

**功能目标**：自动分析每日定时流水线，提供汇总报告和趋势分析

**核心能力**：
- **自动识别**：通过 Workflow 名称模式 + Cron 表达式识别 Nightly 流水线
- **每日汇总**：自动生成 Nightly 构建日报
- **趋势对比**：提供 7 日趋势分析
- **智能告警**：连续失败 2 次触发告警

**日报结构**：
```markdown
# Nightly 构建报告 - 2024-01-15

## 概览
| 指标 | 数值 | 较昨日 |
|------|------|--------|
| 运行总数 | 12 | +2 |
| 成功率 | 85% | +5% |
| 平均时长 | 30 分钟 | -2 分钟 |

## 失败详情
- 基础设施问题 (1)
- 代码问题 (1)

## 7 日趋势
成功率: ████████░░ 85% (↑5%)
```

### 2.3 Weekly 流水线支持（REQ-007）

**功能目标**：分析每周冒烟测试，追踪回归问题，评估整体健康度

**核心能力**：
- **周报生成**：自动汇总一周构建情况
- **回归追踪**：识别本周新增失败，追踪修复进度
- **健康评分**：综合评估系统稳定性
- **目标对比**：对比周目标完成情况

**健康度算法**：
```
健康度评分 = 成功率(40%) + 稳定性(30%) + 效率(20%) + 覆盖率(10%)

评分等级:
  90-100: 🟢 优秀
  80-89:  🟢 良好
  70-79:  🟡 一般
  60-69:  🟡 较差
  0-59:   🔴 危险
```

### 2.4 可视化报表（REQ-008）

**功能目标**：提供多维度可视化数据展示，支持实时监控和历史分析

**核心组件**：

| 组件 | 功能描述 |
|------|---------|
| **实时监控** | 展示当前构建状态、成功率、告警概览 |
| **趋势分析** | 成功率趋势、执行时长趋势、失败数趋势 |
| **分类统计** | 失败原因分布、各分类占比、子类别详情 |
| **健康评分** | 各流水线类型健康度、趋势指示 |
| **失败列表** | 最近失败记录、快速定位、详情跳转 |

**API 接口设计**：

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/v1/dashboard/overview` | GET | 概览数据 |
| `/api/v1/dashboard/trends` | GET | 趋势数据 |
| `/api/v1/dashboard/categories` | GET | 分类统计 |
| `/api/v1/dashboard/recent_failures` | GET | 最近失败 |

### 3.2 数据模型

**时序数据（InfluxDB）**：
```python
measurement: "build_metrics"
tags:
  - pipeline_type: pr/nightly/weekly
  - pipeline_name: str
  - category: code/infrastructure/interference
fields:
  - success: bool
  - duration_seconds: float
  - confidence: float
time: datetime
```

**聚合数据（SQLite）**：
```sql
CREATE TABLE daily_aggregates (
    date DATE NOT NULL,
    pipeline_type TEXT NOT NULL,
    total_runs INTEGER DEFAULT 0,
    success_runs INTEGER DEFAULT 0,
    avg_duration_seconds REAL DEFAULT 0,
    health_score REAL DEFAULT 0,
    UNIQUE(date, pipeline_type)
);
```

## 四、实施计划

### 4.1 阶段划分

| 阶段 | 周期 | 目标 | 交付物 |
|------|------|------|------|
| **Phase 1** | 2 周 | AI 智能分类 | ML 模型服务、准确率 90%+ |
| **Phase 2** | 2 周 | Nightly 支持 | Nightly 分析器、日报生成 |
| **Phase 3** | 2 周 | Weekly 支持 | Weekly 分析器、周报生成 |
| **Phase 4** | 3 周 | 可视化报表 | Dashboard UI、报表 API |
| **Phase 5** | 2 周 | 智能告警 | 多渠道告警、去重策略 |

### 4.2 里程碑验收

**Phase 2：Nightly 分析上线**
- 识别准确率：99%+
- 日报生成：每日自动生成
- 趋势分析：7 日趋势对比
- 告警规则：连续 2 次失败告警

**Phase 3：Weekly 分析上线**
- 周报生成：每周自动生成
- 回归追踪：正确识别新增失败
- 健康度评分：自动计算展示
- 历史对比：与上周数据对比

**Phase 4：可视化仪表盘上线**
- Dashboard 功能：实时监控、趋势分析、分类统计
- 数据刷新：延迟 ≤ 1 分钟
- 响应时间：加载 ≤ 2 秒
- API 覆盖率：完整 RESTful API
